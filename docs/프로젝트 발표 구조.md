# GPTQ-Sinkhorn 프로젝트 발표 구조 (5분)

---

## Slide 1 — Title + Problem (30초)

**제목**: GPTQ-Sinkhorn: Ordering Sinkhorn and Hadamard Rotation for Better LLM Quantization

**한 줄 문제 정의**:
- LLM을 4-bit로 양자화하면 weight 분포의 outlier 때문에 성능이 크게 떨어진다
- 이를 해결하기 위해 Sinkhorn normalization과 Hadamard rotation을 함께 사용하는데, **어떤 순서로 적용해야 할까?**

---

## Slide 2 — Background (45초)

세 기법을 각각 한두 줄로:

- **GPTQ**: Hessian 기반 column-wise 양자화. 이전 column의 quantization error를 이후 column에 전파해 보정
- **Sinkhorn normalization**: 반복적 row/col 정규화로 weight matrix의 행·열 스케일을 균등화 → g(row scale), h(col scale) 추출
- **Hadamard rotation**: 랜덤 부호 벡터 + Walsh-Hadamard transform으로 weight를 회전 → 에너지를 모든 채널에 분산

---

## Slide 3 — Method / Pipeline (30초)

```
W, H = 2·X^T·X / N

  [gh → had]          [had → gh]
  W → Sinkhorn        W → Hadamard
  W → Hadamard        W → Sinkhorn
  → GPTQ              → GPTQ
```

- Sinkhorn 적용 시 H도 h²로 스케일 보정 (수학적으로 일관성 유지)
- 두 순서 모두 구현, ablation으로 비교

---

## Slide 4 — Key Question (20초)

> Sinkhorn과 Hadamard는 **둘 다 outlier를 처리하는 기법**이다.
> 두 기법을 함께 쓸 때, **어떤 것을 먼저 적용해야 더 효과적인가?**

---

## Slide 5 — Theory: 왜 gh→had인가? (60초)

### Hadamard (QuaRot / SpinQuant 계열)
- **한쪽 축(column)만** 회전: `W' = W · diag(d) · U`
- Outlier가 특정 채널에 몰려 있을 때, 에너지를 전체 채널로 **분산**시키지만 **제거하지는 못함**
- 분산된 에너지는 여전히 분포를 heavy-tail로 만듦

### Sinkhorn normalization
- **양쪽 축(row + col) 동시** 정규화
- g, h가 각각 row/col의 스케일 불균형을 **직접 제거**
- 정규화 후 W의 row·col 평균 절댓값이 균등 → outlier 자체가 사라짐

### 결론
```
gh → had:
  Sinkhorn으로 양쪽 축 outlier 제거
  → 이미 균등화된 W를 회전
  → 회전 후 분포가 bell-shape에 근접

had → gh:
  한쪽 축만 섞인 W를 Sinkhorn으로 정규화
  → 이미 퍼진 에너지 위에서 정규화
  → 여전히 heavy-tail 잔존
```

## Slide 8 — Conclusion (20초)

- Sinkhorn과 Hadamard를 **함께** 쓸 때 순서가 성능에 영향을 미친다
- **gh → had**: Sinkhorn으로 양쪽 축 outlier를 먼저 제거 → Hadamard 회전의 효과 극대화
- Weight 분포 시각화 및 통계 지표로 이론적 근거 확인

> "Normalize first, then rotate."
